# ··· TypeScript 编码规范

为了避免重复 review 代码规范，TypeScript 编码规范，已在技术方案中强制格式化，开发者使用时请直接按照步骤安装相关插件，copy 相关规范

# 项目代码格式化方案

核心：eslint + prettier + vscode 自动格式化

# 下载 vscode 插件

  • Prettier
  • ESLint

# 配置 vscode

  • 跟目录下创建.vscode 文件夹
  • 创建 settings.json 文件

向文件中追加

{
  "eslint.validate": [
    "javascript",
    "javascriptreact",
    {
      "language": "typescript",
      "autoFix": true
    },
    {
      "language": "vue",
      "autoFix": true
    }
  ],
  "eslint.autoFixOnSave": true,
  "editor.formatOnSave": true,
  "eslint.enable": true,
  "editor.renderWhitespace": "boundary",
  "editor.cursorBlinking": "smooth",
  "tslint.enable": false,
  "typescript.tsdk": "node_modules/typescript/lib",
  "http.proxy": "http://dev-proxy.oa.com:8080",
  // "vue-peek.targetFileExtensions": [
  // 	".vue",
  // 	".scss",
  // 	".css",
  // 	".png"
  // ],
  "vue-peek.supportedLanguages": ["vue", "typescript"],
  // "path-intellisense.mappings": {
  // 	"@/*": "src/*"
  // },
  "file_peek.activeLanguages": ["typescript", "javascript", "python", "vue"],
  /* 开启保存时自动格式化 */
  "editor.codeActionsOnSave": {
    "source.fixAll.eslint": true
  }
}

# prettier 配置

  • 在根目录下创建 .prettierrc.js
  • 注入以下代码(注释是为了理解，copy 时请删除掉注释)
// .prettierrc.js
{
  // 一行最多 120 字符
  printWidth: 120,
  // 使用 2 个空格缩进
  tabWidth: 2,
  // 不使用缩进符，而使用空格
  useTabs: false,
  // 行尾需要有分号
  semi: true,
  // 使用单引号
  singleQuote: true,
  // 对象的 key 仅在必要时用引号
  quoteProps: 'as-needed',
  // jsx 不使用单引号，而使用双引号
  jsxSingleQuote: false,
  // 末尾需要有逗号
  trailingComma: 'all',
  // 大括号内的首尾需要空格
  bracketSpacing: true,
  // jsx 标签的反尖括号需要换行
  jsxBracketSameLine: false,
  // 箭头函数，只有一个参数的时候，也需要括号
  arrowParens: 'always',
  // 每个文件格式化的范围是文件的全部内容
  rangeStart: 0,
  // 不需要写文件开头的 @prettier
  requirePragma: false,
  // 不需要自动在文件开头插入 @prettier
  insertPragma: false,
  // 使用默认的折行标准
  proseWrap: 'preserve',
  // 根据显示样式决定 html 要不要折行
  htmlWhitespaceSensitivity: 'css',
  // 换行符使用 lf
  endOfLine: 'lf',
};

# eslint 配置

  • 根目录下创建.eslintrc.js
  • 往 js 中注入以下代码
module.exports = {
  root: true,

  env: {
    node: true,
  },

  extends: ['plugin:vue/essential', '@vue/prettier', '@vue/typescript'],

  rules: {
    'no-console': 'off',
    'no-debugger': 'off',
  },

  parserOptions: {
    parser: '@typescript-eslint/parser',
  },

  extends: [
    'plugin:vue/strongly-recommended',
    '@vue/prettier',
    '@vue/typescript',
  ],
};

# ts 命名规范

  1. 使用 PascalCase 为类型命名。
  2. 不要使用 I 做为接口名前缀。
  3. 使用 PascalCase 为枚举值命名。
  4. 使用 camelCase 为函数命名。
  5. 使用 camelCase 为属性或本地变量命名。
  6. 不要为私有属性名添加_前缀。
  7. 尽可能使用完整的单词拼写命名。

# 字符串

  1. 使用单引号''
  2. 字符串拼接使用``
  3. 在句子结尾使用.
  4. 确切的实体应该使用名字（变量名，类型名等）

# 注释

为函数，接口，枚举类型和类使用 JSDoc 风格的注释。
