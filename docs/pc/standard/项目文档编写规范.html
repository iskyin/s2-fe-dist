# ··· 项目文档编写规范

# 文档运行环境

基于 vuepress 构建，相关 API 查看vuepress 文档 (opens new window)

文档目录：docs/

文档运行命令：

# 安装依赖
yarn

# 运行
yarn dev

# 构建
yarn build

# 创建 Markdown 规范

  • 项目文档中在project/文件夹下添加与技术 Markdown 文档
  • 项目文档中在log/文件夹下添加与日志 Markdown 文档
  • 在配置文件.vuepress中的config.js中配置添加相应的文件名
  • 文件名命名尽量简洁

# 核心节点书写规范

  • 指明核心模块绝对路径地址
  • 必须注明开始时间以及完成时间
  • 如果某个版本把此模块迭代掉必须及时更新文档
  • 必须粘贴必要的代码和注释

# 组件文档编写规范

  • 该组件文档必须包含组件属性表
  • 该组件文档必须包含 基础用法
  • 该组件文档必须包含 组件用法实例
